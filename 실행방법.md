# Study Log App 실행 방법

## 📱 Android Studio에서 실행하기

### 1. 프로젝트 열기
1. **Android Studio** 실행
2. `File` → `Open` 선택
3. 프로젝트 폴더 선택: `C:\Users\xcrui\OneDrive\바탕 화면\모프`
4. 프로젝트가 열리면 Gradle 동기화가 자동으로 시작됩니다

### 2. Gradle 동기화
- 자동으로 시작되지 않으면: `File` → `Sync Project with Gradle Files`
- 또는 상단 툴바의 🐘 아이콘 클릭

### 3. Android SDK 확인
- `File` → `Settings` (또는 `Ctrl + Alt + S`)
- `Appearance & Behavior` → `System Settings` → `Android SDK`
- 다음이 설치되어 있어야 합니다:
  - **Android SDK Platform 34** (Target SDK)
  - **Android SDK Build-Tools**
  - **Android SDK Platform-Tools**

### 4. 에뮬레이터 또는 실제 기기 연결

#### 방법 A: 에뮬레이터 사용
1. 상단 툴바의 `Device Manager` 아이콘 클릭
2. `Create Device` 클릭
3. 원하는 기기 선택 (예: Pixel 5)
4. 시스템 이미지 선택 (API 24 이상, 예: API 34)
5. `Finish` 클릭
6. 에뮬레이터 실행

#### 방법 B: 실제 기기 사용
1. Android 기기의 **개발자 옵션** 활성화:
   - `설정` → `휴대전화 정보` → `빌드 번호`를 7번 연속 탭
2. **USB 디버깅** 활성화:
   - `설정` → `개발자 옵션` → `USB 디버깅` 켜기
3. USB 케이블로 PC에 연결
4. 기기에서 "USB 디버깅 허용" 확인

### 5. 앱 실행
1. 상단 툴바에서 실행할 기기 선택 (에뮬레이터 또는 연결된 기기)
2. 녹색 재생 버튼(▶️) 클릭
3. 또는 `Run` → `Run 'app'` (단축키: `Shift + F10`)

### 6. 빌드 오류 발생 시

#### 오류: "SDK location not found"
- `local.properties` 파일이 자동 생성되어야 합니다
- 없다면 프로젝트 루트에 생성하고 다음 내용 추가:
```
sdk.dir=C\:\\Users\\[사용자명]\\AppData\\Local\\Android\\Sdk
```

#### 오류: "Gradle sync failed"
- `File` → `Invalidate Caches / Restart` → `Invalidate and Restart`
- 또는 `Build` → `Clean Project` → `Build` → `Rebuild Project`

#### 오류: "Failed to resolve: androidx..."
- 인터넷 연결 확인
- `File` → `Settings` → `Build, Execution, Deployment` → `Gradle`
- `Use Gradle from` 설정 확인

## 🔧 명령줄에서 실행하기 (선택사항)

### Windows PowerShell에서:
```powershell
# 프로젝트 폴더로 이동
cd "C:\Users\xcrui\OneDrive\바탕 화면\모프"

# Gradle 빌드
.\gradlew.bat assembleDebug

# APK 설치 (기기 연결 필요)
.\gradlew.bat installDebug
```

## 📋 체크리스트

실행 전 확인사항:
- [ ] Android Studio 설치됨
- [ ] Java JDK 설치됨 (Android Studio에 포함)
- [ ] Android SDK 설치됨
- [ ] 최소 API 24 이상의 에뮬레이터/기기 준비
- [ ] 인터넷 연결 (의존성 다운로드용)

## 🎯 첫 실행 후 테스트

1. **앱 실행 확인**: MainActivity가 표시되어야 합니다
2. **캘린더 버튼 클릭**: 캘린더 화면으로 이동
3. **날짜 클릭**: 게시물 작성 화면 열림
4. **사진 추가**: 갤러리 또는 카메라로 사진 추가
5. **저장**: 게시물 저장 후 캘린더에서 확인

## ⚠️ 주의사항

- **권한 요청**: 첫 실행 시 카메라, 저장소 권한을 허용해야 합니다
- **알람 권한**: Android 12 이상에서는 알람 권한을 별도로 요청합니다
- **이미지 저장**: 앱의 내부 저장소에 이미지가 저장됩니다

## 🐛 문제 해결

문제가 발생하면:
1. `Build` → `Clean Project`
2. `Build` → `Rebuild Project`
3. Android Studio 재시작
4. 에뮬레이터/기기 재연결
